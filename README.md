> [!NOTE]
> **LEBOOT** является неофициальным премиальным редизайном (модификацией интерфейса и темы) для аварийного веб-интерфейса U-Boot (Failsafe Web UI) на устройствах MediaTek MT798x.
>
> Проект основан на оригинальном репозитории: [Yuzhii0718/bl-mt798x-dhcpd](https://github.com/Yuzhii0718/bl-mt798x-dhcpd)

>

**LEBOOT** is a premium AMOLED-ready dark theme redesign of the custom U-Boot Failsafe Web UI for MediaTek MT798x by Yuzhii, featuring a beautiful neon orange accent, built-in DHCPD server, advanced environment manager, and a automated zero-touch flashing script.


## 🎨 Описание LEBOOT
**LEBOOT** заменяет стандартный интерфейс восстановления U-Boot на изысканный, современный AMOLED-ready темный интерфейс со стильными неоново-оранжевыми акцентами (`#ff5c00`), плавными микроанимациями и интуитивно понятным разделением настроек на базовые и расширенные разделы.

### Основные улучшения редизайна:
* **AMOLED-ready Dark Theme**: Полностью переработанная темная тема для комфортной работы в ночное время и снижения нагрузки на глаза. Посмотреть как выглядит интерфейс вживую можно в интерактивном симуляторе по адресу [leboot.mudryi.ovh](http://leboot.mudryi.ovh).
* **Премиальная эргономика**: Улучшенная структура боковой панели навигации с красивыми иконками, разделенная на Базовые настройки и Продвинутые инструменты.
* **Поддержка ES6+ на уровне компилятора**: Сборочный конвейер U-Boot переведен с устаревшего минимизатора UglifyJS на современный **Terser**, что позволило использовать полноценный современный JS-код (стрелочные функции, `async/await`, шаблоны строк).
* **Автоматический скрипт прошивки `flash.sh`**: Создан умный консольный скрипт для роутера, который сам проверяет права на запись разделов MTD, автоматически устанавливает обход защиты (`kmod-mtd-rw`) через менеджеры пакетов **`apk`** (современный OpenWrt 24.x+) или **`opkg`** (классический OpenWrt), сам загружает последнюю сборку с GitHub Releases и шьет её в одну команду!

### Полный список возможностей (Features):
- Отображение системной информации (System info)
- Обновление калибровок Factory (RF)
- Скачивание резервных копий (Backup)
- Редактор разделов флеш-памяти (Flash editor)
- Веб-терминал (Web terminal / Console)
- Управление переменными окружения (Environment manager)
- Выбор тем оформления (Theme manager - Bootstrap, GL, MTK, LEBOOT)
- Мультиязычность (I18N: Русский, Украинский, Английский и др.)
- Перезагрузка устройства (Reboot / Reboot to Failsafe)
- Управление томами UBI (UBI volume management)

### Настраиваемые компоненты (Kconfig / Makefile):
- [x] **MTK_DHCPD** — встроенный DHCP-сервер

  - `MTK_DHCPD_POOL_START_HOST` (по умолчанию 100)
  - `MTK_DHCPD_POOL_SIZE` (по умолчанию 101)
- [x] **MTK_TELNETD** — встроенный Telnet-сервер (порт 23)
- Стили аварийного интерфейса (Failsafe Web UI style):
  - [x] `WEBUI_FAILSAFE_UI_BOOTSTRAP` (с поддержкой `WEBUI_FAILSAFE_I18N`)
  - [x] `WEBUI_FAILSAFE_UI_GL`
  - [x] `WEBUI_FAILSAFE_UI_MTK`
  - [x] `WEBUI_FAILSAFE_UI_LEBOOT` (наша кастомная AMOLED-тема)
- [x] **WEBUI_FAILSAFE_ADVANCED** — включить расширенные возможности:
  - [x] `WEBUI_FAILSAFE_SIMG` — поддержка обновления Single Image
  - [x] `WEBUI_FAILSAFE_FACTORY` — обновление калибровок (RF)
  - [x] `WEBUI_FAILSAFE_BACKUP` — скачивание бэкапов

  - [x] `WEBUI_FAILSAFE_CONSOLE` — веб-терминал
  - [x] `WEBUI_FAILSAFE_FLASH` — редактор разделов флеш-памяти
  - [x] `WEBUI_FAILSAFE_UBI` — управление томами UBI

## 🚀 Быстрый запуск и прошивка (в одну команду)

Если у вас уже установлен `ubootmod` и ваш роутер подключен к интернету, вам не нужно ничего скачивать или собирать вручную! 
Просто зайдите на роутер по SSH и выполните следующую команду. Скрипт сам определит ваше устройство, скачает свежую тему с GitHub, разблокирует память ядра и прошьет загрузчик:

```bash
wget -qO /tmp/flash.sh --no-check-certificate https://raw.githubusercontent.com/DarkAssassinUA/LEBOOT/master/flash.sh && sh /tmp/flash.sh
```

## 🛠️ Сборка из исходников

### Требования к среде сборки

```bash
sudo apt install gcc-aarch64-linux-gnu build-essential flex bison libssl-dev device-tree-compiler qemu-user-static nodejs npm
```

> Если вы собираете загрузчик под устройства на базе arm v7l, дополнительно установите `gcc-arm-linux-gnueabi`.

### Инструкция по сборке

1. Сделайте первичную настройку конфигурации:
   ```bash
   make menuconfig
   ```
2. Запустите сборку:
   ```bash
   make
   ```

В `make menuconfig` вы можете гибко настраивать компоненты сборки: `BUILD_FIP`, `BUILD_ATF`, `BUILD_GPT`.

Собрать образы под все поддерживаемые платы можно одной командой:
```bash
make all
```

Для вывода справки по сборке отдельных моделей:
```bash
make help
```

## 🧬 Параметры сборки и варианты разметки

- **Версия сборки (VERSION)**:
  По умолчанию используется `2025` (ATF 20250711 + UBOOT 20250711).
  Также доступны варианты `SP1` (для ядер 5.4) и `SP2` (для новых платформ вроде mt7987 и ядер 6.x).

- **Просмотр поддерживаемых плат**:
  ```bash
  make boards VERSION=2025
  ```

- **Версия сборки (VERSION)** (по умолчанию `2025`):
  
  | Версия | Версия ATF | Версия UBOOT | Описание |
  | --- | --- | --- | --- |
  | **2025** | 20250711 | 20250711 | Основная версия по умолчанию |
  | **SP1** | 20241017-bacca82a8 | 20250711 | Для старых прошивок на ядре 5.4 (исправляет проблемы с hwrng на некоторых устройствах) |
  | **SP2** | 20260123 | 20250711 | Для новых платформ (например, MT7987) и свежих ядер 6.x |

- **Варианты разметки (VARIANT)** (актуально для MTD-устройств):

  | Вариант | Описание | Подходящая прошивка |
  | --- | --- | --- |
  | **default** | Заводская/кастомная разметка с включенным MTK-NMBM | Заводские прошивки и большинство кастомных |
  | **nonmbm** | Заводская/кастомная разметка с отключенным MTK-NMBM | Кастомные прошивки без NMBM |
  | **ubootmod** | Оптимальная совместимость с OpenWrt/ImmortalWrt | Прошивки с ubi/ubootmod разметкой |
  | **ubi** | Специальный UBI-лейаут (например, `spi-nand0:1024k(bl2),-(ubi)`) | Прошивки с UBI-разметкой |
  | **openwrt** | Официальная разметка OpenWrt (без веб-интерфейса аварийного восстановления) | Чистый образ OpenWrt |

## 🖲️ Управление кнопкой Failsafe Web UI

По умолчанию кнопка входа в веб-интерфейс настроена как `glbtn_key=reset,wps,mesh` (скрипт ищет первые доступные GPIO с этими именами).
Вы можете переопределить это поведение вручную из консоли U-Boot:
* Указать конкретный GPIO: `setenv glbtn_gpio 0`
* Изменить полярность сигнала кнопки: `setenv glbtn_gpio !0`
* Назначить кнопку по её имени: `setenv glbtn_key wps`

### Дополнительные параметры сборки (`make [options]`):

| Параметр | Тип | Обязательный | По умолчанию | Описание |
| --- | --- | --- | --- | --- |
| **SOC** | string | Нет | автоопределение | Платформа процессора (например, `mt7981`, `mt7986`) |
| **MULTI_LAYOUT** | boolean | Нет | 1 | `MULTI_LAYOUT=0` отключает поддержку мульти-разметки (только NAND) |
| **FIXED_MTDPARTS** | boolean | Нет | 1 | `FIXED_MTDPARTS=0` делает разметку mtdparts редактируемой |
| **FSTHEME** | string | Нет | leboot | Тема оформления веб-интерфейса (`bootstrap`/`gl`/`mtk`/`leboot`) |
| **SIMG** | boolean | Нет | 0 | `SIMG=1` активирует поддержку Single Image обновления |
| **UBIMNG** | boolean | Нет | 0 | `UBIMNG=1` включает управление томами UBI в веб-интерфейсе |
| **TELNETD** | boolean | Нет | 0 | `TELNETD=1` активирует сервер Telnet (порт 23) в аварийном режиме |
| **CLEAN** | boolean | Нет | null | Запуск с ключом `--clean` для очистки окружения перед сборкой |

*(После изменений не забудьте сохранить их командой `saveenv` и перезагрузить устройство).*
